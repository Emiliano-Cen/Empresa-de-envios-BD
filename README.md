Sistema de Base de Datos – Empresa de Logística

- Proyecto académico desarrollado en el curso de Base de Datos 1, enfocado en el diseño, modelado e implementación de una base de datos para una empresa de transporte y logística.
- El objetivo fue representar correctamente la realidad del negocio mediante modelos conceptuales y relacionales, asegurando integridad, consistencia y eficiencia en el manejo de los datos.

---

Tecnologías utilizadas

- SQL
- Modelado de datos
- Diseño de bases de datos relacionales

---

Características principales

- Diseño completo del sistema de datos desde cero
- Modelado mediante:
  - Modelo Entidad-Relación (MER)
  - Modelo relacional
- Definición de entidades, atributos y relaciones
- Implementación de:
  - Scripts DDL (creación de tablas)
  - Scripts DML (inserción y manipulación de datos)
- Aplicación de:
  - Normalización
  - Restricciones de integridad
- Resolución de consultas mediante SQL

---

Conceptos aplicados

- Modelado conceptual (MER)
- Transformación a modelo relacional
- Normalización de bases de datos
- Claves primarias y foráneas
- Integridad referencial
- Relaciones:
  - Uno a muchos (1:N)
  - Muchos a muchos (N:N)
  - Relaciones recursivas
- Consultas SQL

---

Objetivo del proyecto

Diseñar una base de datos robusta que represente fielmente las operaciones de una empresa de logística, permitiendo gestionar información de manera eficiente y consistente.

---

Cómo utilizar

1. Ejecutar los scripts SQL en el gestor de base de datos correspondiente
2. Crear las tablas y estructuras definidas
3. Insertar datos de prueba
4. Ejecutar consultas para validar el funcionamiento

---

Contenido del repositorio

- Script de creación de base de datos (DDL)
- Script de carga de datos (DML)
- Consultas SQL
- Diagramas (MER y modelo relacional)
